The problem

At Afresh, design inconsistencies were mounting as the system aged. The existing library, created a year prior, was confusing and underutilised, consuming time and compromising quality.

The goals

The team needed a shared, up-to-date library that would increase speed and make components consistent.

Research

I conducted eight interviews with team members and facilitated a workshop. Key findings: colleagues struggled with the system, doubted its reliability, and felt confused by it. Contributing guidelines were unclear, foundational rules were missing, and tech debt created misaligned expectations.

A few ideas

Audit the existing system, make templates, and define foundations: colour, spacing, and typography.

Auditing

I audited the product suite to identify gaps between the design system and live products.

Designing

I created documentation templates, developed a new type scale, implemented Figma tokens for colours, and documented icon library standards.

Image
Image

Then there was a problem

Documentation proved difficult to maintain and was rarely read, so the team abandoned it. Instead, we prioritised building really nice components that would be easy to grab and use.

Final

Strong components paired with solid typography, grid, and colour foundations increased library adoption and simplified updates.
